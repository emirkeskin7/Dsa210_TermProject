# Predicting Formula 1 Mechanical DNFs (2014–2025)

DSA 210 — Spring 2026 term project.

## Motivation

In the high-stakes world of Formula 1, mechanical reliability is a strategic
component as important as raw pace. A car failing to finish a race (a
mechanical DNF) is rarely pure luck — it tends to follow from cumulative
environmental stress, track characteristics, and component fatigue. This
project investigates whether mechanical DNFs in the modern hybrid era are
predictable from circuit-level and seasonal factors, and how strong each of
those predictors actually is.

## Repository contents

| File | Purpose |
|------|---------|
| `main.py` | Data-collection script. Pulls every F1 race result from 2014 to 2025 via FastF1, enriches each row with circuit altitude and circuit type, labels mechanical DNFs, and writes `f1_term_project_data.csv`. |
| `f1_term_project_data.csv` | The prepared dataset: one row per driver per race (≈ 4,860 rows). |

## Reproducing the project

```bash
# 1. Set up the environment
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 2. (Re)build the dataset from FastF1.  Takes ~10 minutes the first time;
#    subsequent runs use the local cache and are much faster.  Skip this step
#    if you already have f1_term_project_data.csv.
python main.py

# 3. Run the analysis
pip install jupyterlab
jupyter lab F1_Mechanical_DNF_Analysis.ipynb
# … or open the .ipynb in VS Code and "Run All".
```

## Hypotheses tested

The notebook tests four hypotheses about what predicts mechanical DNFs:

1. **H1 (altitude):** higher-altitude circuits should see more failures
   (thinner air, hotter engines).
2. **H2 (circuit type):** street circuits should see more failures than
   permanent racetracks (walls, kerbs, surface).
3. **H3 (seasonal fatigue):** failures should accumulate as the season
   wears on (more kilometres on every component).
4. **H4 (engine lifecycle):** failures should cluster late in each
   engine's planned life, using the FIA per-season power-unit quota
   (5 PU/driver in 2014, 4 in 2015–17, 3 in 2018–22, 4 in 2023–25).

The notebook also adds a machine-learning section (logistic regression
and random forest) on the same feature set, with class-balanced training
and ROC-AUC as the headline metric.

Two methodological corrections vs. an earlier draft of the analysis are
documented inside the notebook: H1 is now tested at the circuit level
(altitude is a circuit-level variable, not a per-row one), and H4 is
tested using the actual FIA quota rather than a fixed mod-7 cycle.

## Data sources

* **Race results (2014–2025):** the [FastF1](https://docs.fastf1.dev/)
  Python library, which wraps the official F1 timing data.
* **Circuit enrichment (altitude, type):** hand-curated lookup table in
  `main.py`, drawn from public circuit information.
